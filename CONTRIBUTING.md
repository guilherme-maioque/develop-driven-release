To ensure our automated release system works correctly, all contributors must follow the **Conventional Commits** standard.

## **Commit Message Format**

Each commit must follow this structure:  
\<type\>\(scope\): \<description\>

### **Allowed Types:**

* feat: A new feature for the user. (Increments **Minor** version)  
* fix: A bug fix for the user. (Increments **Patch** version)  
* perf: Code change that improves performance.  
* docs: Changes to documentation only.  
* chore:, style:, refactor:, test: Technical changes that do not affect the end-user (Hidden from Changelog).
